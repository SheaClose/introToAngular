SPA Framework

SPA stands for Single Page Application. This means the entire Application takes place on a single URL.
This is awesome because:
  • No more page re-loads during standard navigation and for EVERY user interaction.
  • More responsive - Page reacts similarly to native application.
  • Increased performance - because the HTML is sent in the initial request, only data is requested from servers, decreasing load time for each state change.
Negatives:
  • Increased initial load time. Because we are loading everything up front, the initial load time is slightly increased.
  • SEO - Until very recently, search engines like Google and Bing had a difficult time crawling and indexing Single Page Applications.

MVC

MVC Stands for Model View Controller. MVC is an architectural concept that is not specific to Angular.
MVC:
  • The Model represents the business logic of your app
  • The View is the app as displayed to the user.
  • The Controller is what handles the modification of the model and the view based on user interaction. We will have access to the Controller from the DOM via "$scope".

$scope and Data-Binding

Scope is an object that refers to the application model. Scopes can watch expressions and propagate events.
$scope:
  • $scope is JUST an object. There is nothing magic about it. It is an object that will have properties added to it (including methods), which are made accessible to the DOM via Data-Binding.
  • Scope is the glue between application controller and the view.
  • Both controllers and directives have reference to the scope, but not to each other. This is an important point since it makes the controllers view agnostic.
Data-Binding:
  • Data-binding is the automatic synching of data between the model and view.
  • This lets you treat the model as the single-source-of-truth. The view is a projection of the model at all times.
  • When the model changes, the view reflects the change, and vice versa.

Angular Directives

Directives are very powerful, and handle the majority of the work for us. They allow us to establish links between the View and the Model. Directives have the ability to execute methods, define behavior, attach controllers and $scope objects, manipulate the DOM, and more.
Directives:

  • ng-app

    - It is added to set the AngularJS section. Any HTML element that is a child of this directive will be part of our Application. Should be set as a high-level Element (HTML/Body).

  • ng-controller

    - It is used to attach a controller class to the view. This attribute sets which controller (and thus which $scope) our data is bound to.

  • ng-click

    - It is used to execute a method or expression when element is clicked. Most commonly used on buttons, though it can be added to ANY HTML element.

  • ng-style

    - It is used to set CSS style on an HTML element conditionally.
