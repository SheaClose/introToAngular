Angular vs React

Both are SPA frameworks, but where as React takes a uni-directional and 'functional' approach, Angular is all about that MVC.
    Angular is very opinionated. Much of the functionality is abstracted out. This can make Angular a little more difficult to learn because things must be done the 'Angular' way, but once you learn it, the experience is very Dev friendly.

MVC

MVC Stands for Model View Controller. MVC is an architectural concept that is not specific to Angular.
MVC:
  • The Model represents the business logic of your app
  • The View is the app as displayed to the user.
  • The Controller is what handles the modification of the model and the view based on user interaction. We will have access to the Controller from the DOM via "$scope".

$scope and Data-Binding

Scope is an object that refers to the application model. Scopes can watch expressions and propagate events.
$scope:
  • $scope is JUST an object. It is an object that will have properties added to it similar to a components state, but with other neat functionality.
  • Scope is the glue between application controller and the view.
  • Both controllers and the directives on the view have reference to the scope, but not to each other. This is an important point since it makes the controllers view agnostic. (This is less important in component based architecture.
Data-Binding:
  • Data-binding is the automatic synching of data between the model and view.
  • This lets you treat the model as the single-source-of-truth. The view is a projection of the model at all times. If any user interaction modifies the model, the view is automatically update to reflect the change.
  • When the model changes, the view reflects the change, and vice versa.

Angular Directives

Directives are very powerful, and handle the majority of the work for us. They allow us to establish links between the View and the Model. Directives have the ability to execute methods, define behavior, attach controllers and $scope objects, manipulate the DOM, and more.
Directives:

  • ng-app

    - It is added to set the AngularJS section. Any HTML element that is a child of this directive will be part of our Application. Should be set as a high-level Element (HTML/Body).

  • ng-controller

    - It is used to attach a controller class to the view. This attribute sets which controller (and thus which $scope) our data is bound to.

  • ng-click

    - It is used to execute a method or expression when element is clicked. Most commonly used on buttons, though it can be added to ANY HTML element.

  • ng-model

    - It is used to bind an input, select, text area etc. to a property on the $scope.
    • <input type="text" ng-model="myInput">
    <li> Display for ng-model: { { myInput } } </li>

  • ng-repeat

    - It is used to loop through each item in collection to create a new template.
    • $scope.items = ["cup", "bottle", "mouse", "desk", "computer", "chair"]

    <ul ng-repeat="item in items">

    <li> { { item } } </li>

    </ul>

  • ng-show/ng-hide

    - It works based on expression, if true then the element is shown or hidden respectively.

    • <li ng-hide="show">

  • ng-if

    - It is used to remove or recreate an element in the DOM depending on an expression. This differs from show/hide in that with show/hide, the element is on the DOM, but is set to "display: none". With ng-if, if the provided expression does not evaluate to true, the element is not rendered to DOM at all.
