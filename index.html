Why Angular?

Jquery's code quickly becomes complicated and hard to read. Harder to maintain
Angular is:
  • Modular- Due to seperation of concerns, you can keep your components small, focused, and thus, reusable.
  • DRY - Since you can reuse your code, you do not constantly repeat yourself.
  • Readable - No more spaghetti code, everything in it's place.

SPA Framework

SPA stands for Single Page Application. This means the entire Application takes place on a single URL.
This is awesome because:
  • No more page re-loads during standard navigation and for EVERY user interaction.
  • More responsive - Page reacts similarly to native aplication.
  • Increased performance - becuase the HTML is sent in the initial request, only data is requested from servers, decreasing load time for each state change.
Negatives:
  • Increased intial load time. Because we are loading everything up front, the initial load time is slightly increased.
  • SEO - Until very recently, search engines like Google and Bing had a difficult time crawling and indexing Single Page Applications.

MVC

MVC Stands for Model View Controller. MVC is an architectual concept that is not specific to Angular.
MVC:
  • The Model represents the business logic of your app
  • The View is the app as displayed to the user.
  • The Controller is what handles the modification of the model and the view based on user interaction. We will have access to the Controller from the DOM via "$scope".

Angular Directives

Directives are very powerful, and handle the majority of the work for us. They allow us to establish links between the View and the Model. Directives have the ability to execute methods, define behavior, attach controllers and $scope objects, manipulate the DOM, and more.
Directives:
  • ng-app - It is added to set the AngularJS section. Any HTML element that is a child of this directive will be part of our Application. Should be set as a high-level Element (HTML/Body).
  • ng-controller - It is used to attach a controller class to the view. This attribute sets which controller (and thus which $scope) our data is bound to.
  • ng-click - It is used to execute a method or expression when element is clicked. Most commonly used on buttons, though it can be added to ANY HTML element.
  • ng-style - It is used to set CSS style on an HTML element conditionally.
  • ng-model - It is used to bind an input,select, textarea etc to a property on the $scope.
  • Diplay for ng-model: {{myInput}}
  • ng-repeat - It is used to loop through each item in collection to create a new template.
  • ng-show/ng-hide - It workes based on expression, if true then the element is shown or hidden respectively.
  • ng-if - It is used to remove or recreate an element in the DOM depending on an expression. This differs from show/hide in that with show/hide, the element is on the DOM, but is set to "display: none". With ng-if, if the provided expression does not evaluate to true, the element is not rendered to DOM at all.
